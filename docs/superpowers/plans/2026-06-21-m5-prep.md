# M5-prep Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Close the five locally-verifiable M0–M4 loose ends (dashboard uncleared KPI, `is_system`
read guard, bank-account deactivation with an uncleared-guard, project subagents, Node pin) and reframe
the two deployment-gated items in the TODO.

**Architecture:** Backend is a .NET modular monolith (CQRS via a hand-rolled `ISender`, minimal-API
endpoints, EF Core + raw SQL reads, Postgres RLS). Cross-module reads go through consumer-owned
**batch** ports (ADR-007) implemented by thin host adapters. The SPA is React + TS (Vite, TanStack
Query) over a generated OpenAPI client. This plan adds one shared read field (`UnclearedCount`), wires
it into the dashboard, adds a guarded bank-deactivation command + a new clearance port, hardens the
`is_system` filter behind a funnel + behavioral test, authors agent files, and bumps the Node pin.

**Tech Stack:** .NET 10, EF Core/Npgsql, PostgreSQL 18, FluentValidation, xUnit v3 + Shouldly +
Testcontainers, React 19 + TypeScript + Vite, Vitest + MSW, Playwright.

## Global Constraints

- **Money is `decimal` in C# / `NUMERIC(14,2)` in Postgres. Never float/double.**
- **Append-only journal:** the runtime role has no UPDATE/DELETE on `journal_entries`/`journal_lines`/
  `audit_events`. This plan writes none of those; `bank_account.is_active` and `bank_line_status` are
  not append-only.
- **RLS is the boundary:** every org-scoped read runs on the ambient scoped connection; no new
  connections. No new org-scoped table is added (so no RLS-helper migration is needed).
- **Cross-module rule (ADR-007):** a module never reads another module's tables/types; cross-module
  reads go through a consumer-owned **batch** port (returns a map) + a host adapter delegating via
  `ISender`. `ModuleBoundaryTests` enforces the assembly half.
- **No MediatR, no AutoMapper. Minimal APIs only (`TypedResults`), no MVC controllers.** RFC 7807
  ProblemDetails for errors (`TypedResults.Problem`).
- **Zero schema migrations expected** — every column touched (`bank_accounts.is_active`,
  `bank_line_status.status`) already exists. If a step adds a migration, stop and re-check.
- **Node 26** is the CI/CLAUDE pin (`.github/workflows/ci.yml` `node-version: '26'`).
- **Definition of Done (per task):** tests at the right altitude green in CI; empty/loading/error +
  keyboard path + AA/icon-label (not color alone) on new UI; demoable on the seeded demo org with no
  manual fixes; `dotnet format --verify-no-changes --exclude …/Migrations` and `npm run format:check`
  clean; `web/src/api/schema.d.ts` regenerated with no drift when the API surface changes.
- **Branch:** all work lands on `m5-prep` (already created off `main`); one commit per task.

```csharp
using LeaseBook.Modules.Directory.Domain;
using LeaseBook.SharedKernel.Cqrs;
using Microsoft.EntityFrameworkCore;

namespace LeaseBook.Modules.Directory.Features.BankAccounts;

/// <summary>
/// Activates or deactivates a bank account (M2-E9). Reactivation is always allowed; deactivation is
/// blocked while the account has uncleared items (checked through the Accounting clearance port,
/// ADR-007) so an inactive account never hides outstanding work. Deactivation only removes the account
/// from new-posting pickers (ListBankAccounts(ActiveOnly: true)); read/reconcile surfaces are unchanged.
/// </summary>
public sealed record SetBankAccountActive(Guid Id, bool IsActive) : ICommand<SetActiveResult>;

public enum SetActiveOutcome
{
    Updated,
    NotFound,
    BlockedUncleared,
}

public sealed record SetActiveResult(SetActiveOutcome Outcome, BankAccountResponse? Bank);

internal sealed class SetBankAccountActiveHandler(DbContext db, IBankClearanceStatus clearance)
    : ICommandHandler<SetBankAccountActive, SetActiveResult>
{
    public async Task<SetActiveResult> Handle(SetBankAccountActive command, CancellationToken ct)
    {
        var bank = await db.Set<BankAccount>().FirstOrDefaultAsync(b => b.Id == command.Id, ct);
        if (bank is null)
        {
            return new SetActiveResult(SetActiveOutcome.NotFound, null);
        }

        if (!command.IsActive)
        {
            var uncleared = await clearance.UnclearedCountsAsync(ct);
            if (uncleared.GetValueOrDefault(bank.Id) > 0)
            {
                return new SetActiveResult(SetActiveOutcome.BlockedUncleared, null);
            }
        }

        bank.IsActive = command.IsActive;
        await db.SaveChangesAsync(ct);
        return new SetActiveResult(SetActiveOutcome.Updated, BankAccountResponse.From(bank));
    }
}
```
